# Жизненный цикл Issue

## Поток

1. **Ideation** — идейная формулировка
2. **Ready** — уточнение scope + AC + DoD
3. **In Progress** — выполнение задачи по коммитам
4. **Review** — проверка в PR
5. **Done** — после прохождения DoD

## Критические правила

- Ни один Task не закрывается без обязательного отчёта.
- Без DoD — не закрывать в GitHub.
- Обновление `docs/roadmap.md` — обязательная часть DoD.
